# 준비 중 지표 9개 raw-first + API-refresh 구현 계획

## 1. 목표

현재 시스템은 다음 2단계 구조를 전제로 한다.

1. **초기 구축:** 원자료를 다운로드하여 `data/raw/`에 보존하고, SQLite DB에 적재하여 앱/정제 파이프라인의 기준 데이터로 활용한다.
2. **갱신:** 공공데이터포털/대학알리미 OpenAPI 등으로 최신 연도 데이터를 업데이트한다.

이 문서는 준비 중 지표 9개에 대해 원자료 확보 출처, 계산식, 표준화 기준, DB 적재 구조, API 갱신 방향, `registry/metadata.py` 호환 제안을 정리한다.

## 2. 공통 데이터 처리 원칙

### 2.1 원자료 보존

- 다운로드 원본은 가공하지 않고 `data/raw/<source>/<dataset>/<year>/` 아래 보존한다.
- 파일명은 가능하면 원문 파일명을 유지하되, 중복 방지를 위해 다운로드 일자를 접두/접미로 붙일 수 있다.
- 원자료 등록 메타데이터는 DB의 `raw_sources` 테이블에 남긴다.

권장 경로 예시:

```text
data/raw/academyinfo/student_status/2025/original.xlsx
data/raw/academyinfo/education_condition/2025/original.xlsx
data/raw/kasfo/financial_indicator/2025/original.xlsx
data/raw/kosaf/scholarship/2025/original.csv
```

### 2.2 표준화 키

모든 raw 적재/정제 결과는 최소한 아래 키를 갖도록 맞춘다.

| 표준 컬럼 | 설명 |
|---|---|
| `school_name` | 학교명. 원자료 표기를 보존하되 비교용 정규화 컬럼 별도 생성 가능 |
| `school_id` | 대학알리미 학교 ID. 원자료에 없으면 후속 매핑 테이블로 보강 |
| `campus` | 본교/분교/캠퍼스 구분 |
| `founding_type` | 국립/공립/사립 등 설립유형 |
| `school_type` | 대학/전문대학/대학원대학 등 |
| `region` | 지역명 |
| `year` | 공시년도/회계연도/기준연도. 지표별 의미를 `year_type`으로 구분 가능 |
| `value` | 최종 지표값 |
| `numerator` | 계산형 지표의 분자 |
| `denominator` | 계산형 지표의 분모 |
| `unit` | %, 명, 원, ㎡ 등 |
| `source_id` | `raw_sources.source_id` 참조 |

### 2.3 SQLite 테이블 초안

```sql
CREATE TABLE raw_sources (
  source_id TEXT PRIMARY KEY,
  source_name TEXT NOT NULL,
  source_url TEXT NOT NULL,
  provider TEXT NOT NULL,
  dataset_name TEXT NOT NULL,
  acquired_method TEXT NOT NULL,
  acquired_at TEXT,
  local_path TEXT,
  license_note TEXT,
  auth_required INTEGER NOT NULL DEFAULT 0,
  notes TEXT
);

CREATE TABLE raw_records (
  source_id TEXT NOT NULL,
  table_name TEXT NOT NULL,
  row_number INTEGER NOT NULL,
  payload_json TEXT NOT NULL,
  PRIMARY KEY (source_id, table_name, row_number)
);

CREATE TABLE metric_values (
  metric_id TEXT NOT NULL,
  school_name TEXT NOT NULL,
  school_id TEXT,
  campus TEXT,
  founding_type TEXT,
  school_type TEXT,
  region TEXT,
  year INTEGER NOT NULL,
  year_type TEXT NOT NULL,
  value REAL,
  numerator REAL,
  denominator REAL,
  unit TEXT NOT NULL,
  source_id TEXT NOT NULL,
  calculated_at TEXT NOT NULL,
  PRIMARY KEY (metric_id, school_name, campus, year, year_type)
);
```

### 3.7 재학생 1인당 강의실 면적

- **metric_id 제안:** `classroom_area`
- **dataset_key 제안:** `classroom_area`
- **우선 원자료:** 공공데이터포털 `한국대학교육협의회_대학알리미 교육여건 현황`
- **확인된 데이터:** `https://www.data.go.kr/data/15037506/openapi.do`
- **정의 확인 출처:** 한국대학평가원 `[중요] 4주기 대학기관평가인증 평가 기준 확정본 및 평가 일정 안내`의 `붙임2. 4주기 대학기관평가인증 평가 기준.pdf`, 양식 4.4-1 및 평가준거 4.4.
- **확보 가능성:** 중간~높음. 교육여건 현황에서 일반 강의실/멀티미디어 강의실 면적과 재학생 수 원자료 확보 여부 확인 필요.
- **계열 구분 여부:** 계열별 산출 지표가 아니다. 일반 강의실과 멀티미디어 강의실 면적 합계를 전체 재학생 수로 나눈다. 캠퍼스가 분리된 경우 캠퍼스별 작성이 필요하다.
- **계산식:**

```text
재학생 1인당 강의실 면적 = (일반 강의실 면적 + 멀티미디어 강의실 면적) / 재학생 수
```

- **분자:** 일반 강의실 면적 + 멀티미디어 강의실 면적(㎡). 복도 등 공용 면적은 제외.
- **분모:** 재학생 수. 학부+대학원, 정원내+정원외 기준이며, 주야간이 동시에 있는 학부 학과는 재학생 수가 많은 쪽을 사용한다. 계약학과, 특성화고 졸업 재직자 특별전형 학과, 야간 및 계절제 대학원은 제외한다.
- **단위:** `㎡`
- **평가기준값:** `1.2㎡`
- **API-refresh:** 교육여건 현황 API + 학생 현황 API 조합.
- **registry series 제안:**
  - `id="classroom_area_per_student"`
  - `label="재학생 1인당 강의실 면적"`
  - `column="재학생1인당강의실면적"`
  - `unit="㎡"`
  - `decimals=2`

### 3.8 재학생 1인당 실험실습실 면적

- **metric_id 제안:** `lab_area`
- **dataset_key 제안:** `lab_area`
- **우선 원자료:** 공공데이터포털 `한국대학교육협의회_대학알리미 교육여건 현황`
- **정의 확인 출처:** 한국대학평가원 `붙임2. 4주기 대학기관평가인증 평가 기준.pdf`, 양식 4.4-2 및 평가준거 4.4.
- **확보 가능성:** 중간~높음. 교육여건 활용가이드에서 계열별 실험·실습실 면적 항목 확인 필요.
- **계열 구분 여부:** 계열별 산출이 필요하다. 구분은 `인문사회계열`, `자연과학계열`, `공학계열`, `의학계열`, `예체능계열`이다.
- **계산식:**

```text
계열별 재학생 1인당 실험실습실 면적 = 계열별 실험·실습실 면적 / 계열별 재학생 수
평균 = 전체 실험·실습실 면적 / (인문사회계열 재학생 수 × 1/3 + 나머지 계열 재학생 수)
```

- **분자:** 계열별 실험·실습실 면적(㎡). 복도 등 공용 면적은 제외.
- **분모:** 계열별 재학생 수. 평균 산출 시 인문사회계열 재학생 수는 `1/3` 가중치를 적용한다.
- **단위:** `㎡`
- **평가기준값:** `2.5㎡`
- **작성 주의:** 여러 계열이 공동 사용하는 실험·실습실은 계열별 재학생 수 기준으로 배분한다. 대학원 실험·실습실도 포함한다. 캠퍼스가 분리된 경우 캠퍼스별 작성이 필요하다.
- **API-refresh:** 교육여건 현황 API + 학생 현황 API 조합.
- **registry series 제안:**
  - `id="lab_area_per_student"`
  - `label="재학생 1인당 실험실습실 면적"`
  - `column="재학생1인당실험실습실면적"`
  - `unit="㎡"`
  - `decimals=2`

### 3.9 재학생 1인당 실험실습 기자재 구입비

- **metric_id 제안:** `lab_equipment`
- **dataset_key 제안:** `lab_equipment`
- **우선 원자료:** 대학알리미 재정 현황 또는 교육여건 현황
- **확인된 관련 API 후보:** `한국대학교육협의회_대학알리미 재정 현황` (`https://www.data.go.kr/data/15038392/openapi.do`)
- **정의 확인 출처:** 한국대학평가원 `붙임2. 4주기 대학기관평가인증 평가 기준.pdf`, 양식 4.4-3 및 평가준거 4.4.
- **확보 가능성:** 중간. 대학알리미/대학재정 원자료에서 계열별 `실험·실습 기자재 구입비`와 계열별 재학생 수를 함께 확보할 수 있는지 확인 필요.
- **계열 구분 여부:** 계열별 산출이 필요하다. 작성 대상은 `자연과학계열`, `공학계열`, `의학계열`, `예체능계열`이다. `인문사회계열`은 기준값 산출 참고용으로 사용한다.
- **계산식:**

```text
재학생 1인당 실험실습 기자재 구입비 = 최근 2년 실험·실습 기자재 구입비 합계 / 최근 2개 학년도 재학생 수 합계
기준값 = min(해당 대학 자연과학계열-인문사회계열 평균 등록금 차액 × 15%, 전국 기준값)
```

- **분자:** 2024~2025회계연도 결산 기준 계열별 실험·실습 기자재 구입비. 기계·기구 매입비, 실험·실습 기자재/집기 구입비, 리스 임차료를 포함하고 시약·샘플 구입비는 제외한다.
- **분모:** 2024~2025학년도 계열별 재학생 수 합계.
- **단위:** `천원`
- **작성 주의:** 자연과학계열이 없는 경우 공학계열 등록금 기준을 사용한다. 자연과학계열과 공학계열이 모두 없으면 정성평가 대상이다.
- **API-refresh:** 재정 현황 API 또는 원자료 파일 + 학생 현황 API 조합.
- **registry series 제안:**
  - `id="lab_equipment_expense_per_student"`
  - `label="재학생 1인당 실험실습 기자재 구입비"`
  - `column="재학생1인당실험실습기자재구입비"`
  - `unit="천원"`
  - `decimals=0`

## 4. 구현 우선순위

### 4.1 1차 구현 후보

원자료 확보 가능성과 계산식 명확성을 기준으로 아래 4개를 1차 구현 후보로 둔다.

1. `student_recruitment` — 학생 현황 API/파일에서 신입생·재학생 충원율 직접 확보 가능성이 높음
2. `staff_per_student` — 재학생 수와 직원총계로 계산 가능
3. `scholarship_ratio` — 대학재정알리미 장학금 지급률 또는 장학재단 파일로 확보 가능
4. `corp_transfer_ratio` — 대학재정알리미 법인전입금 비율 직접 지표 가능성이 높음

### 4.2 2차 구현 후보

1. `classroom_area`
2. `lab_area`
3. `lab_equipment`
4. `adjunct_faculty`
5. `corp_finance_ratio`

2차 후보는 원자료의 세부 컬럼/정의 확인이 먼저 필요하다.

## 6. 주의사항

- 공공데이터포털 OpenAPI는 활용신청/서비스키가 필요할 수 있다.
- 대학재정알리미는 지표 다운로드 자동화 가능 여부를 별도로 확인해야 한다.
- 학교명만으로 join하지 말고 가능한 경우 학교 ID를 우선한다.
- 본교/분교 기준, 회계연도/공시년도 기준이 지표별로 다르므로 `year_type`을 반드시 남긴다.
- 계산식이 확정되지 않은 지표는 앱에서 `implemented=True`로 바꾸지 않는다.
