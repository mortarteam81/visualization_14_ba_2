# 1차 구현 후보 4개 원자료 확보 경로 확정 메모

## 목적

준비 중 지표 9개 중 1차 구현 후보 4개에 대해 `raw-first + API-refresh` 구조에 맞는 실제 확보 경로, 파일/API 형식, 인증 필요 여부, 구현 리스크를 정리한다.

1차 구현 후보:

1. 학생 충원 성과 (`student_recruitment`)
2. 직원 1인당 학생수 (`staff_per_student`)
3. 장학금 비율 (`scholarship_ratio`)
4. 세입 중 법인전입금 비율 (`corp_transfer_ratio`)

## 요약 결론

| 지표 | raw-first 확보 경로 | 형식 | 인증 | API-refresh 경로 | 우선 구현 판단 |
|---|---|---|---|---|---|
| 학생 충원 성과 | 교육부_대학알리미_대학주요정보 XLSX + 학생 현황 원자료 | XLSX/XML | 파일은 인증 없음 또는 웹 다운로드, API는 서비스키 필요 | 대학정보공시 학생 현황 API | 1순위 |
| 직원 1인당 학생수 | 교육부_대학알리미_대학주요정보 XLSX | XLSX | 인증 없음 또는 웹 다운로드 | 필요 시 학생/직원 현황 API 조합 | 1순위 단, 직원총계 컬럼 확인 필요 |
| 장학금 비율 | 대학재정알리미 장학금 지급률 또는 한국장학재단 장학금 수혜 CSV | CSV/웹 지표 | 파일 다운로드는 인증 없음, API는 서비스키/활용신청 필요 | 한국장학재단 JSON/XML 또는 재정 현황 API | 1순위 |
| 세입 중 법인전입금 비율 | 대학재정알리미 대학재정 주요 지표 | 웹 지표/다운로드 | 웹 조회, 다운로드 방식 확인 필요 | API 미확인 | 1순위이나 다운로드 자동화 확인 필요 |

## 1. 학생 충원 성과

### 1.1 raw-first 후보

#### 후보 A: 교육부_대학알리미_대학주요정보_학생_교원_연구_재정_교육여건

- 공공데이터포털 페이지: `https://www.data.go.kr/data/15118998/fileData.do`
- 제공기관: 교육부
- 원 출처 URL: `https://www.academyinfo.go.kr/main/main2130/doInit.do`
- 형식: XLSX
- 비용: 무료
- 이용허락: 공공저작물 출처표시 제1유형
- 제공 데이터 중 확인된 관련 항목:
  - 공시년도
  - 학교명
  - 본분교명
  - 학교종류
  - 학교유형
  - 설립유형
  - 지역명
  - 입학정원(학부)
  - 재학생(학부)
  - 신입생경쟁률(학부)
  - 신입생 충원율(학부)

#### 후보 B: 한국대학교육협의회 대학정보공시 학생 현황

- 공공데이터포털 페이지: `https://www.data.go.kr/data/15037346/openapi.do`
- 형식: XML REST API
- 업데이트 주기: 실시간
- 활용승인: 개발/운영 자동승인
- 트래픽: 개발계정 1,000
- 인증: `ServiceKey` 필수
- 확인된 관련 항목:
  - 신입생 충원율
  - 재학생 충원율
  - 재학생수
  - 재적학생수
  - 졸업자 수
  - 취업자 수

### 1.2 계산식

원자료에 충원율 값이 있으면 직접값을 우선 사용한다.

```text
신입생 충원율 = 입학자 수 / 모집인원 × 100
재학생 충원율 = 재학생 수 / 편제정원 × 100
```

### 1.3 구현 판단

- 최초 raw DB 구축은 XLSX 또는 대학알리미 다운로드 원자료를 우선한다.
- API는 최신 연도 업데이트에 사용한다.
- `학생 충원 성과`를 단일값으로 만들지 말고 `신입생 충원율`, `재학생 충원율` 2개 series로 구현하는 것이 안전하다.

## 2. 직원 1인당 학생수

### 2.1 raw-first 후보

#### 후보 A: 교육부_대학알리미_대학주요정보_학생_교원_연구_재정_교육여건

- 공공데이터포털 페이지: `https://www.data.go.kr/data/15118998/fileData.do`
- 형식: XLSX
- 확인된 관련 항목:
  - 재학생(학부)
  - 직원총계는 검색 결과/대학알리미 주요지표 설명에서 확인되나, XLSX 실제 컬럼 존재 여부는 다운로드 후 검증 필요

### 2.2 계산식

```text
직원 1인당 학생수 = 재학생 수 / 직원총계
```

### 2.3 구현 판단

- 원자료에서 `직원총계`가 실제 제공되면 1차 구현 가능.
- 만약 `대학주요정보` XLSX에 직원총계가 없으면 대학알리미 직원 현황 원자료를 별도로 확보해야 한다.
- 분자/분모 모두 학교명·본분교·공시년도 기준으로 join해야 한다.

## 3. 장학금 비율

### 3.1 raw-first 후보

#### 후보 A: 대학재정알리미 장학금 지급률

- 페이지: `https://uniarlimi.kasfo.or.kr/statistics/indicator`
- 지표명: 장학금 지급률
- 정의: 학생부담 등록금 대비 장학금 지급 비율
- 형식: 웹 지표/대학별 상세 조회
- 다운로드 방식: 사이트 내 엑셀/상세 다운로드 가능 여부 추가 확인 필요

#### 후보 B: 한국장학재단_대학별 장학금 수혜 현황

- 공공데이터포털 페이지: `https://www.data.go.kr/data/15038576/fileData.do`
- 제공기관: 한국장학재단
- 형식: CSV 파일 + 자동변환 XML/JSON API
- 업데이트 주기: 연간
- 차기 등록 예정일: 2026-10-31
- 파일 다운로드: 로그인 없이 다운로드 가능하다고 명시
- API 활용: 공공데이터포털 회원가입 및 활용신청 필요
- 확인된 관련 항목:
  - 교내장학금
  - 교외장학금
  - 재원구분
  - 성격유형
  - 기준연도/조사연도

### 3.2 계산식

대학재정알리미 직접 지표를 쓰는 경우:

```text
장학금 지급률 = 학생부담 등록금 대비 장학금 지급 비율
```

한국장학재단 원자료로 재계산하는 경우:

```text
장학금 비율 = 장학금 지급액 / 등록금 수입 × 100
```

### 3.3 구현 판단

- 앱 지표명이 `장학금 비율`이고 설명이 `등록금 수입 대비 장학금 지급 비율`이므로 계산형 정의를 유지할 수 있다.
- 다만 대학재정알리미의 `장학금 지급률`은 분모가 `학생부담 등록금`이므로 현재 앱 설명과 정확히 일치하는지 확인 필요.
- 초기 구현은 대학재정알리미 직접 지표 또는 한국장학재단 CSV 중 하나를 선택해야 한다.

## 4. 세입 중 법인전입금 비율

### 4.1 raw-first 후보

#### 후보 A: 한국사학진흥재단 대학재정알리미 대학재정 주요 지표

- 페이지: `https://uniarlimi.kasfo.or.kr/statistics/indicator`
- 지표명: 법인전입금 비율
- 분야: 법인책무성
- 정의: 법인의 대학에 대한 재정기여도
- 해설: 비율이 높을수록 운영수입이 다양하게 구성되어 등록금 의존율이 낮아지며, 법인의 대학에 대한 재정적 책무성 정도가 높은 것을 의미
- 형식: 웹 지표/대학별 상세 조회
- 다운로드 방식: 엑셀 다운로드 가능 여부 추가 확인 필요

### 4.2 계산식

```text
법인전입금 비율 = 법인전입금 / 운영수입 × 100
```

### 4.3 구현 판단

- 직접 지표가 있으므로 구현 가능성은 높다.
- 다만 자동 수집 전 대학재정알리미 화면의 조회 파라미터/다운로드 요청 방식을 확인해야 한다.
- API가 없다면 Playwright 또는 수동 다운로드 파일 기반 raw 적재로 시작하는 것이 안전하다.

## 5. 구현 순서 제안

### 5.1 1차 스프린트

1. `student_recruitment`
   - 공공데이터포털 학생 현황 API/대학주요정보 XLSX로 신입생·재학생 충원율 확보
2. `staff_per_student`
   - 대학주요정보 XLSX에서 `재학생`, `직원총계` 실제 컬럼 검증
3. `scholarship_ratio`
   - 한국장학재단 CSV 구조 확인
4. `corp_transfer_ratio`
   - 대학재정알리미 다운로드 방식 확인

### 5.2 구현 전 확인해야 할 것

- `data.go.kr` 서비스키를 프로젝트에서 사용할지 여부
- 대학재정알리미 다운로드 자동화 허용 여부
- `장학금 비율`의 공식 분모를 `등록금 수입`으로 유지할지, 대학재정알리미 `학생부담 등록금` 기준으로 맞출지
- `학생 충원 성과`를 단일 성과값으로 합성할지, 2개 series로 유지할지

## 7. 2026년 대학주요정보 공개 화면 데이터 확보 결과

### 7.1 확인한 실제 엔드포인트

대학알리미 `대학주요정보 한눈에 보기` 화면은 RealGrid 기반이며, 엑셀 다운로드 버튼은 브라우저 로컬 export 방식이다. 서버에서 바로 XLSX를 내려받는 고정 다운로드 URL은 확인되지 않았다. 대신 화면 데이터는 아래 공개 엔드포인트에서 JSON으로 조회 가능함을 확인했다.

```text
POST https://www.academyinfo.go.kr/main/main2130/selectSchlList.do
Referer: https://www.academyinfo.go.kr/main/main2130/doInit.do
form data:
  svyYr=<공시연도>
  schKndCdArr=
  schEstbDivCdArr=
  schZnCdArr=
  schZnCdArr1=
  numStu=
```

### 7.2 2026년 원자료 저장 위치

아래 파일을 생성했다.

```text
data/raw/academyinfo/university_key_indicators/2026/academyinfo_key_indicators_2026.json
data/raw/academyinfo/university_key_indicators/2026/academyinfo_key_indicators_2026.csv
```

- 조회 건수: 489건
- 인증: 없음
- API 서비스키: 필요 없음
- 성격: 대학알리미 공개 화면 데이터

### 7.3 student_recruitment 후보 processed 파일

아래 후보 파일을 생성했다.

```text
data/processed/student_recruitment/student_recruitment_2026_candidate.csv
```

포함된 핵심 컬럼:

- 공시연도
- 대표학교코드
- 학교코드
- 학교명
- 본분교명
- 대학구분
- 학교종류
- 설립유형
- 지역명
- 학교상태
- 입학정원(학부)
- 재학생수(학부)
- 신입생경쟁률(학부)
- 신입생충원율(학부)
- 신입생충원율(학부)_기준년도

주의:

- `재학생충원율`은 이 엔드포인트에서 제공되지 않는다.
- 따라서 `student_recruitment`를 완전 구현하려면 공공데이터포털 `한국대학교육협의회 대학정보공시 학생 현황` API 또는 학생 현황 원자료에서 재학생 충원율을 추가 확보해야 한다.

### 7.4 staff_per_student 확인 결과

대학주요정보 공개 화면 데이터에는 `재학생수(학부)`는 있으나 `직원총계`는 확인되지 않았다.

따라서 `직원 1인당 학생수 = 재학생 수 / 직원총계` 구현을 위해서는 아래 중 하나가 추가로 필요하다.

1. 대학알리미 직원 현황 원자료
2. 직원총계가 포함된 별도 대학주요정보 다운로드 파일
3. 공공데이터포털/대학알리미에서 직원 현황 API 또는 파일데이터 확보

## 8. 2026년 raw DB 적재 결과

`data/raw/pending_metrics.sqlite`에 계획 메타데이터와 2026년 대학주요정보 raw CSV를 적재했다.

```text
pending_metric_plans: 9
raw_sources: 10
raw_records: 489
metric_values: 0
```

아직 `metric_values`는 계산/확정 단계 전이므로 비워두었다.
