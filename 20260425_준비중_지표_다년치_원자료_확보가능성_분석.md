# 준비중 지표 다년치 원자료 확보 가능성 분석

조사일: 2026-04-25

## 결론

기존 구현 방향처럼 2025년 단년 자료가 아니라 확보 가능한 최대 연도 자료로 DB를 구축할 수 있다. 우선순위는 다음과 같다.

1. 대학알리미 지표는 `공시데이터 추이`를 1순위로 사용한다. 대표 파일 다운로드를 검증했고, `4-라-1. 재학생 충원율` 파일은 2008~2025년 데이터를 포함한다.
2. 대학재정알리미/KASFO 지표는 `대학재정데이터`와 공공데이터포털 파일데이터를 함께 사용한다. 법인·교비 결산은 2015~2024, 교육용 건물 세부현황은 2014~2025까지 확보 가능성이 확인됐다.
3. 4주기 대학기관평가인증 기준의 4.4 교육시설 지표 중 `실험·실습실 면적`과 `실험·실습 기자재 구입비`는 계열별 계산이 필요하다. 다만 공공 원자료는 계열별 완성 지표가 아니라 건물/회계 원장에 가깝기 때문에, 계열 배분 로직 또는 대학 내부 제출 양식 수준의 보완 자료가 필요하다.
4. API는 대안으로 유효하지만, 이번 조사 기준으로는 파일 다운로드 방식이 더 적합하다. 대학알리미와 KASFO 모두 다년치 원자료 파일을 직접 확보할 수 있기 때문이다.

## 확인한 기존 패턴

- 기존 `dormitory_accommodation_status_v2`는 대학알리미 `공시데이터 추이` XLSX를 받아 CSV/DB로 정규화한다.
- 기존 `education_cost_return_rate`는 대학재정알리미/KASFO 계열 파일을 다년치로 모아 long format으로 정규화한다.
- 따라서 새 지표도 `연도`, `학교코드/학교명`, `본분교`, `설립`, `지역`, `계열/회계/캠퍼스` 같은 차원 컬럼을 보존하고, 원자료별 스키마를 명시한 뒤 DB 적재하는 방식이 맞다.

## 수집 경로 검증 결과

### 대학알리미 공시데이터 추이

- URL: https://www.academyinfo.go.kr/main/main0830/main0830.do
- 페이지 설명상 항목별 주요지표 추이를 위한 연도별 공시데이터와 원자료를 제공한다.
- 목록 API: `/main/main0830/selectGongsiDataList.do`
- 다운로드 흐름:
  - `/popup/main2120/doInsert.do`로 다운로드 사유 등록
  - `/main/main0830/FileDown.do` 호출
  - `file_nm`에는 화면 표시 파일명, `file_dtl_nm`에는 실제 `CMN81_###.xlsx` 파일명을 넣어야 한다.
- 검증 파일:
  - `data/raw/pending_manual/academyinfo_trend_samples/academyinfo_trend_student_fill_rate_CMN81_015.xlsx`
  - `4-라-1. 재학생 충원율_대학_정원내 재학생 충원율.xlsx`
  - 포함 연도: 2008~2025

### 대학알리미 공시항목 단년 다운로드

- 공시데이터 추이에 없는 항목도 `/popup/main0810/selectReqList.do`, `/popup/main0810/selectReqRst.do`, `/popup/main0810/download.do` 흐름으로 연도별 ZIP 생성이 가능하다.
- `14-사. 직원 현황`은 공시데이터 추이 목록에는 없지만, 2008, 2013, 2014, 2020, 2021, 2022, 2023, 2024, 2025 조회가 성공했다.
- 검증 파일:
  - `data/raw/pending_manual/academyinfo/academyinfo_2024_195_staff_status_school.zip`
  - 직접 XLSX URL 접근은 404/오류 페이지가 반환되므로 ZIP 생성 API를 사용해야 한다.

### 대학재정알리미/KASFO 대학재정데이터

- URL: https://uniarlimi.kasfo.or.kr/knowledge/data2Room
- 확인/확보 파일:
  - `data/raw/pending_manual/facilities/kasfo_education_buildings_detail_2014_2023.xlsx`
  - `data/raw/pending_manual/facilities/kasfo_education_buildings_detail_2021_2025.xlsx`
  - `data/raw/pending_manual/finance/kasfo_private_university_corporate_school_settlement_2015_2020.zip`
  - `data/raw/pending_manual/finance/data_go_kasfo_private_university_corporate_school_settlement_db_5yr_20250531.zip`
  - `data/raw/pending_manual/finance/kasfo_corporate_general_and_school_account_settlement_2024.xlsx`
- 2025년 11월 7일 게시된 `최근 10개년 국·공·사립대학 교사 현황('21~'25)` 및 과거 `('14~'23) 교육용 건물 세부현황`으로 시설 데이터는 2014~2025까지 커버 가능하다.
- 2024회계연도 법인일반회계 및 교비회계 결산은 2025년 9월 16일 게시되어 있다. 2025회계연도는 현재 본예산 파일은 보이나 결산 파일은 확인되지 않았다.

### 공공데이터포털 KASFO 파일/API 대안

- `한국사학진흥재단_최근 5개년도 사립대학 법인 교비 결산서 DB`: https://www.data.go.kr/data/15125334/fileData.do
  - CSV ZIP 다운로드 가능
  - 기준: 2024회계연도 결산 완료 공개 시점
  - 차기 등록 예정일: 2026-09-30
- `한국사학진흥재단_사립대학 법인일반 및 교비 자금계산서`: https://www.data.go.kr/data/15135558/fileData.do
  - 전입금수입, 기부금수입, 인건비, 관리운영비 등을 학교별/법인별로 제공한다고 명시
  - XML/JSON 자동변환 API가 제공되지만 활용신청이 필요하다.

## 4주기 기준에서 주의할 점

한국대학평가원 `4주기 대학기관평가인증 평가 기준`은 2024년 12월 20일 확정본으로 게시되어 있고, 4.4 교육시설 기준은 다음 조건을 포함한다.

- 강의실 면적: 2026년 4월 1일 기준, 일반 강의실과 멀티미디어 강의실 합계 면적을 사용한다.
- 실험·실습실 면적: 인문사회, 자연과학, 공학, 의학, 예체능 계열별로 작성하며, 인문사회계열 재학생 수는 1/3로 계산한다.
- 실험·실습 기자재 구입비: 자연과학, 공학, 의학, 예체능 계열별로 작성하고, 2024~2025회계연도 결산 기준의 최근 2년 평균을 사용한다.
- 기자재 구입비에는 `기계·기구매입비`, 실험·실습실 집기·비품 매입비, 리스 임차료 등을 포함한다.

따라서 공공 원자료로 바로 구현 가능한 것은 `총량 기반 시계열`이고, 4주기 기준과 동일한 `계열별 인증 산식`은 다음 보완이 필요하다.

- 실험·실습실 면적: 건물/실 단위 원자료를 계열에 배정하는 기준
- 기자재 구입비: 회계 지출을 계열별로 배분하거나 대학 내부 제출 양식 수준의 계열별 파일 확보
- 강의실 면적: 멀티미디어 강의실을 별도 분리할 수 있는 원자료 확보 여부

## API 대안

파일 원자료 확보가 어려울 때의 대안은 다음 순서로 검토한다.

1. 대학알리미 OpenAPI
   - 학생 현황: https://www.data.go.kr/data/15037346/openapi.do
   - 교원·연구 현황: https://www.data.go.kr/data/15037505/openapi.do
   - 교육여건 현황: https://www.data.go.kr/data/15037506/openapi.do
   - 재정 현황: https://www.data.go.kr/data/15038392/openapi.do
   - 장점: 학교ID/공시년도 기준의 구조화 데이터
   - 한계: 서비스키 필요, 원자료 상세 컬럼보다 지표값 중심일 수 있음
2. 공공데이터포털 파일데이터 자동변환 API
   - KASFO 법인일반 및 교비 자금계산서처럼 CSV 파일을 XML/JSON API로 자동변환 제공
   - 장점: 원본 CSV와 동일 계열 데이터를 API로 수집 가능
   - 한계: 활용신청 필요, 다운로드 파일이 더 단순할 수 있음

## 추천 후속 작업

1. 원자료 수집 스크립트 분리
   - `scripts/fetch_academyinfo_trend_files.py`: 공시데이터 추이 목록 조회 및 `CMN81_###.xlsx` 다운로드
   - `scripts/fetch_academyinfo_item_year_files.py`: 직원 현황처럼 추이에 없는 항목을 연도별 ZIP으로 생성/다운로드
   - `scripts/fetch_kasfo_data_room_files.py`: KASFO data2Room 첨부파일 manifest 기반 다운로드
2. raw schema 등록
   - 대학알리미 trend 파일, 대학알리미 item-year ZIP, KASFO 법인교비 ZIP/CSV, KASFO 건물 세부현황 XLSX를 각각 스키마로 등록한다.
3. 구현 우선순위
   - 1차: 학생 충원 성과, 직원 1인당 학생수, 장학금 비율
   - 2차: 법인전입금 계열 2개 지표
   - 3차: 강의실/실험실습실/기자재 지표의 총량 기반 버전
   - 4차: 계열별 배분 기준 확정 후 4주기 산식 버전
